✅ Before You Start — You Need

  • A GitHub account ✅ (you already have one!)
  • Your KTCM project pushed to GitHub (we'll do this first)
  • A Railway account (sign up with GitHub — takes 30 seconds)
💡 Railway connects directly to your GitHub repo and deploys automatically every time you push changes!
1
Push Project to GitHub First
Open VS Code terminal in your ktcm folder and run:
git init git add . git commit -m "KTCM E-Vote Production System" git branch -M main git remote add origin https://github.com/YOURUSERNAME/ktcm-evote.git git push -u origin main
✅ Your code is now on GitHub — Railway will pull from here
2
Create Railway Account
  1. Go to railway.app
  2. Click "Login with GitHub"
  3. Authorize Railway to access your GitHub
  4. You're in! Railway dashboard opens
✅ Signing in with GitHub is the easiest method
3
Create New Railway Project
  1. Click "New Project" on Railway dashboard
  2. Select "Deploy from GitHub repo"
  3. Find and select your ktcm-evote repository
  4. Railway will detect the Dockerfile automatically
  5. Click "Deploy Now"
💡 Railway reads your Dockerfile and builds PHP + Apache automatically — no manual server setup needed!
4
Add MySQL Database Plugin
  1. In your Railway project click "+ New"
  2. Select "Database"
  3. Choose "MySQL"
  4. Railway creates a MySQL database and connects it automatically
  5. Click on the MySQL service → go to "Variables" tab
  6. You will see: MYSQLHOST, MYSQLUSER, MYSQLPASSWORD, MYSQLDATABASE, MYSQLPORT
✅ Your config.php already reads these variables automatically — no changes needed!
5
Import Database via Railway MySQL
  1. Click on your MySQL service in Railway
  2. Go to the "Data" tab
  3. Click "Connect" → copy the connection string
  4. Open TablePlus or MySQL Workbench on your PC
  5. Connect using Railway's MySQL credentials
  6. Run the contents of database_complete.sql
💡 Alternative: Use Railway's built-in query editor — paste the SQL from database_complete.sql and run it
6
Get Your Live URL
  1. Click on your PHP/Apache service in Railway
  2. Go to "Settings" tab
  3. Under "Domains" click "Generate Domain"
  4. You get a free URL like: ktcm-evote.up.railway.app
🗳️ Student Portal: https://ktcm-evote.up.railway.app/index.html ⚙️ Admin Panel: https://ktcm-evote.up.railway.app/admin.html
✅ HTTPS is automatic — your system is secure out of the box!
7
Hash All PINs on Live Server
  1. Visit: https://ktcm-evote.up.railway.app/hash_generator.php
  2. Wait for ✅ green success messages
  3. Go to GitHub → delete hash_generator.php from the repo
  4. Railway will auto-redeploy without it
⚠️ Always delete hash_generator.php after running it on a live server!

🔄 Auto-Deploy (Best Feature!)

Every time you push code changes to GitHub, Railway automatically redeploys your site. No manual uploading!

# Make a change in VS Code, then: git add . git commit -m "Updated candidate list" git push # Railway detects the push and redeploys in ~2 minutes ✅
✅ This means you can manage the system from anywhere — just push to GitHub!

  • Generate a QR code from your Railway URL at qr.io and share with students
